# 📊 Tableau DataFest 2025

`#Tableau` `#TableauNext` `#시각화` `#BI운영` `#지표표준화`

> 📅 **2025.08.27** · 🏢 **Salesforce** · 📍 파르나스 인터콘티넨탈

## 🔑 Key Findings

> - **태블로의 다양한 상품들을 알 수 있고, 특히 태블로 경진대회(태블로 시각화를 eSports화)를 재미있게 보았음**
>   - 특히 Tableau Next가 향후 나아갈 제품의 방향인 것 같은데, 모든 제품들이 이렇게 빠르게 AI를 붙여 나오는 트랜드가 인상적이었음
> - **(별도로 신청한) 태블로 닥터는 Sailing Stone 업체에서 나왔으며 정말 전문가여서 도움이 많이 되었음.** 나중에 필요하면 협업하면 좋을 것 같음
> - **강의는 30분으로 짧기도 했는데**, 발표자 회사를 홍보하고 태블로를 도입하기까지의 얘기가 대부분이어서 크게 와 닿는 내용은 없었음 (통계를 통한 마케팅 효과 상승이나 활용까지는 안 감)
> - **컨텍포인트를 만들어 놓았기 때문에 추후 미팅을 도모할 수 있을 것으로 보임**
>   - Tableau Next: AI를 덧붙인 Tableau로 영업팀에 연락처 남겼고, 필요하면 회사로 와서 상품 피칭해주겠다고 함
>   - Sailing Stone도 연락처 교환하였으며, Tableau 관련 문의를 할 수 있음 – 단, 난이도에 따라서 비용이 들어갈 수 있음

## 💡 Lessons Learned

- **태블로 닥터의 실전 팁은 바로 적용 가능** — 유료 결제자 '신규/유지/이탈' 분석은 Raw data를 UserKey·결제일·최초결제일 형태로 추출하고 계산된 필드로 처리하라. 그래야 그룹 선택 후 추가 분석(ARPU 구간·결제 게임·참여 이벤트)이 가능하다.
- **컨텍포인트 2건 확보** — Tableau Next POC 피칭, Sailing Stone 컨설팅(넷마블 게임 데이터 프로젝트 경험). 필요할 때 연결 가능.
- **모든 BI 제품에 AI가 붙는 속도** — Tableau Pulse·Agent·Next까지, 지표를 공통 정의해 사내 기준을 만드는 방향이 공통 흐름.

<summary>Tableau Doctor — Sailing Stone 1:1 컨설팅 (20분)</summary>

- 닥터는 Sailing Stone이라고 태블로 시각화 컨설팅 업체에서 해주었음 (Salesforce와 관련이 있다고 함)
- 넷마블에서도 프로젝트해서 게임 데이터 봤다고 함
- 필요한 경우 향후 컨설팅(1회도) 가능: 컨텍 포인트 확보했음. 단, 비용 있음
- 원천 데이터 합치는 것 관련 문의했는데, 메일로 관련 자료 준다고 하셨음. 메일로 문의하면 무료일 수도 있지 않을까 함

- 닥터 상담 내용
  - 질의: 유료 결제자 추이 — 전월 대비 '신규', '유지', '이탈' 수를 보고 싶음. 이 경우 Raw data를 어떤 형식으로 추출해서 연결하면 될지?
    - RAW data를 다음과 같이 추출하라: UserKey / 결제일 / 최초결제일 / 상품, 금액… — 나머지 내용들은 계산된 필드로 해서 하라
    - 결제일은 리포트에서 '월'만 남기고 Userkey 'distinct'로 계산하게 하라
    - '신규', '유지', '이탈'은 계산된 필드를 만들어서 사용하면 된다
    - 이렇게 해야 나중에 특정 그룹 선택 시 추가 분석이 가능하다 — 해당 그룹의 ARPU 구간별, 결제 게임, 참여 이벤트 등등
    - 더 나아가 해당 유저들을 대시보드에서 선택하면 해당 유저들의 특성을 보고 싶음 (유저 트래킹)
  - 질의: 월간 통계 정리 — Streaming data 말고 정제 데이터를 연결하여 리포트를 만들 수 있을지?
    - 이벤트 정보를 정리한 데이터 + Streaming 데이터 특정 기간으로 한정해서 사용
    - 정제된 엑셀 파일도 올려서 Streaming 데이터와 합쳐질 수 있다. 이 부분에 대해서 교육 자료를 보내주겠음
  - 질의: 다른 리포트에서 사용하고 있는 테이블을 다른 테이블이랑 합쳐서 사용할 수 있는지? — 현재 태블로에 있는 리포트를 여러 개 합쳐서 하나로 만든 후 Visualization하고 싶음 (공헌이익/거래액/유저 참여자 수 + 이벤트 성과)
    - 오늘 문의한 내용 관련된 교육 자료 보내주기로 함
  - 그 외
    - Raw data는 제대로 갖고 있어야 분석을 쉽게 할 수 있다. 컬럼 20-30개 x row 8000개 정도는 되어야 데이터가 많은 거다
    - 데이터가 너무 많아서 버벅거리면 쪼개서 일자별 카테고리별이건 보는 것이 좋다
    - Data Creator가 있으면 좋다. 이게 있으면 있는 데이터 여러 개 그냥 join해서 사용하면 된다. (하지만 비용은 2배가 될 거다.)
    - Userkey를 보고 싶은데 개인정보 이슈가 있다면 계산된 필드에서 마스킹해서 (right *) 내보내면 된다

   - 사용자들을 고려한 사용성 높은 서비스 기획이 필요
   - 전사 단위로 대시보드 확장 시 단계별 확장 권장

   - 해당 주제가 유효한 주제인지 사전에 선별 필요

<summary>Come to User, Com2us Tableau — 조의찬 대리, 컴투스 데이터전략실</summary>

1. 회사 소개
   - 컴투스에서 Tableau 시각화 업무를 하는 업무를 맡고 있음
   - 게임 데이터의 특징은 방대, 실시간으로 수집, 다차원적이라는 것임
     - 게임 데이터는 모든 행동을 로그로 남길 수 있기 때문에 무궁무진함 (한 테이블이 몇천억 방대함, 실시간 트래킹도 가능함) 그래서 다차원 로그가 있음
   - 그리하여 효율적인 데이터 조회 전략 & 효과적인 BI 시스템의 구축이 중요함
2. 데이터 시각화하는 소재들
   - 컴투스에서는 보통 리텐션을 많이 보고 있음
   - 긴 기간 서비스하는 게임들은 트랜드만 빠르게 보고 싶어함 (증감)
   - 데이터로 매출 현황 트래킹+BM 상품 구성에도 활용함 → 매출 성장에 기여
   - 이탈 시점도 봄
3. 데이터팀으로서 실무자들 지원
   - 사내교육
   - BI 알림봇
   - 사용자 지정뷰, 구독 메일링
4. Tableau Pulse(하나의 지표를 집중해서 봄), Tableau Agent 서비스를 활용하고 있음
5. 태블로 리포트 사례
